map()

Dentre as diversas funes do pacote purrr uma das mais conhecidas e utilizadas a map() e suas variaes, nada mais justo do que comear falando dela. Essa funo (e suas variaes) usada para aplicar a mesma ao/funo a todos elementos de um objeto. Esses objetos podem ser uma lista, um vetor, variveis de um banco de dados, etc. Esse tipo de funo muito til para evitar usar loops e fors.

Se em algum momento voc j fez uso da famlia de funes apply() voc pode ter percebido uma certa semelhana entre elas. Existem algumas discusses entre o uso das funes apply() e map(). No o foco deste post fazer uma comparao aprofundada sobre essas funes, mas podemos trazer alguns pontos. De forma geral, escolher qual delas usar vai depender de um contexto. Em questo de tempo bruto de execuo, a famlia apply() ligeiramente mais rpida (a diferena bem pequena), porm as funes de map() so mais convenientes. Todas as suas variaes seguem a mesma ordem de argumentos (o primeiro argumento sempre o vetor), diferente da famlia apply(), onde os argumentos podem mudar. Existem mais pontos a serem considerados, mas em resumo, se ignorar questes de sintaxe e funcionalidade (as variaes de map()), decidir qual delas usar restringe-se a usar as funes base do R ou utilizar funes obtidas por meio de pacotes, neste caso o purrr.

Voltando ao foco, vamos mostrar algumas variaes (no todas) de map().

  • map(.x, .f) a principal funo de mapeamento e retorna uma lista.

  • map_df(.x, .f) retorna um data.frame.

  • map_dbl(.x, .f) retorna um vetor numrico (duplo).

  • map_chr(.x, .f) retorna um vetor de caracteres.

  • map_lgl(.x, .f) retorna um vetor lgico.

Essas funes, assim como a maiores das funes do universo tidyverse, tem como primeiro argumento o objeto a qual voc deseja aplicar uma certa funo e o segundo argumento a funo a ser aplicada em cada elemento do seu objeto. Para ficar mais claro vamos a uma breve aplicao e ao decorrer do post, iremos utilizar mais as outras funes. Vamos criar um vetor com poucos elementos para usar como exemplo, depois criar uma funo qualquer e usar o map() para aplicar essa funo em cada elemento do vetor.

vetor_ex <- c(0, 3, 6, 9, 12)

funcao1 <- function(x){
  return(x^2)

Nos exemplos acima utilizamos uma funo criada por ns e outras j presentes dentro do R. Alm dessas, tambm possvel utilizar o que chamamos de funes annimas. Uma funo annima basicamente uma funo criada e usada, mas nunca atribuda a um objeto. Esse tipo de funo bem til dentro do map(). Para usar essa funo voc comea com um ~ (para indicar que voc comeou uma funo annima). O argumento da funo consultado utilizando o .x ou apenas ., diferente das funes normais onde voc consegue usar qualquer argumento. Para ficar mais claro vamos a um exemplo.

map_dbl(vetor_ex, ~ .x ^2)
## [1]   0   9  36  81 144
vetor_ex %>% 
  map_dbl(~ .x ^2)
## [1]   0   9  36  81 144
vetor_ex %>% 
  map_dbl(~ . ^2)
## [1]   0   9  36  81 144
vetor_ex %>% 
  map_dbl(~{.x ^2})
## [1]   0   9  36  81 144

Acima mostramos maneiras diferentes de utilizar funes annimas. Todas retornam o mesmo resultado (e deveriam). Um destaque para a ltima onde foi usado {}. O uso das chaves interessante para organizao e boas prticas no seu cdigo, principalmente se a sua funo for mais complexa.

Uma funo muito til tambm a modify(). Ela funciona de forma igual funo map(), a diferena que ela sempre ir retornar um objeto do mesmo tipo que o objeto de entrada. Por exemplo, se meu objeto de entradar um vetor de caracteres, a sada tambm ser um vetor de caracteres. Note que isso pode ser considerado como uma boa prtica e j fizemos um post sobre isso no software R disponvel aqui.

objeto <- data.frame(x = c(1, 10, 11),
                     y = c(9, 20, 30))

modify(objeto, funcao1)
##     x   y
## 1   1  81
## 2 100 400
## 3 121 900

A funo modify() possui uma variao interessante. A modify_if() permite que voc fornea um critrio e assim, a funo s ser aplicada aos elementos que atendam a esse critrio. O critrio torna-se o segundo argumento da funo, ou seja, o primeiro argumento continua sendo o objeto de entrada, o segundo passa a ser o critrio e o terceiro argumento a funo. Para ficar mais claro vamos a um exemplo

modify_if(vetor_ex,
          ~ . >= 9,
          funcao1)
## [1]   0   3   6  81 144

Veja que a funo s foi aplicada para valores maiores ou iguais a 9.

Vamos comear a fazer algo mais elaborado. Podemos querer retornar algumas medidas resumos ou outras informaes gerais sobre nossos dados em uma base de dados. map2()

Aps obtermos uma certa noo de como funcionam as funes de map() podemos falar de sua outra verso, a map2(). Essa funo segue os mesmos critrios que a map() original, porm permite que voc trabalhe no s com um, mas com dois objetos. Em map2() tambm conseguimos especificar o tipo de sada da funo, segue abaixo a lista de algumas das variaes da funo.

  • map2 (.x, .y, .f, ...)

  • map2_dfc (.x, .y, .f, ...)

  • map2_dbl (.x, .y, .f, ...)

  • map2_chr (.x, .y, .f, ...)

  • map2_lgl (.x, .y, .f, ...)

Lembrando que .x e .y so os vetores da funo e devem ter o mesmo tamanho, caso contrrio a funo retornar uma mensagem de erro.

dados3 <- dados %>% 
  group_by(Sepal.Width, Species) %>% 
  summarise(n = n())
## `summarise()` has grouped output by 'Sepal.Width'. You can override using the `.groups` argument.
largura <- dados3$Sepal.Width

especie <- dados3$Species

fmp2 <- function(x, y) paste("Espcie:", x, "largura:", y)

map2(especie, largura, fmp2) %>% head(5)

Acima criamos dois vetores, cada um contendo uma varivel do banco de dados e depois criamos uma funo simples, que ir concatenar, nesse caso, os dois vetores com algumas frases. Note que a funo foi aplicada aos dois vetores. Uma observao que no comeo do exemplo criamos um outro banco de dados (dados3) com apenas as duas variveis que foi usada no exemplo e agrupamos elas utlizando group_by() esummarise(). Isso foi feito apenas para que fosse possvel ver os resultados com observaes diferentes na saida da funo.

possvel utilizar o map() para mais de dois objetos. Neste caso no existe um map3() ou map4() mas sim o pmap(). pmap() basicamente uma generalizao de map2() para 3 ou mais objetos. Essa funo possui uma pequena diferena, nesse caso voc no ir especificar n objetos dentro da funo, mas sim apenas uma nica lista que contm todos os vetores (ou listas).

keep(), discard() e compact()

A funo keep() e discard() so bem parecidas e opostas. keep() mantm apenas os elementos de uma lista que satisfazem uma determinada condio, enquanto discard() faz exatamente o oposto (descarta quaisquer elementos que satisfaam a condio lgica). Essas funes possuem 2 argumentos principais, .x uma lista ou vetor e .p uma funo. Apenas aqueles elementos .p avaliados como TRUE sero mantidos ou descartados. Tambm possvel, como terceiro argumento, fornecer argumentos adicionais transmitidos para .p. compact() uma funo com argumentos parecidos com as anteriores, a diferena que para essa funo, o .p uma funo que aplicada a cada elemento de .x e somente aqueles elementos .p avaliados como um vetor vazio (NULL ou listas com comprimento zero) sero descartados.

every(), some() e none()

Caso a gente queira saber se todos, alguns ou nenhum dos elementos de uma lista satisfazem um determinado critrio, podemos usar as funes every(), some() e none(), respectivamente. Essas funes so bsicas e possuem apenas dois argumentos, sendo o primeiro uma lista ou vetor, e o segundo argumento o critrio. A sada da funo ser valores lgicos (TRUE ou FALSE).

dados4 %>% 
  every(~{mean(.x$Petal.Width) > 0.5})
## [1] FALSE
dados4 %>% 
  some(~{mean(.x$Petal.Width) > 0.5})
## [1] TRUE
dados4 %>% 
  none(~{mean(.x$Petal.Width) > 20})
## [1] TRUE

Aqui podemos puxar um gancho para uma outra fuo de pesquisa que tambm retorna um valor lgico, has_element. Basicamente serve para testar se uma lista ou vetor contm um determinado objeto, a funo tem o seguinte formato:

  • has_element(.x, .y)

onde .x a lista ou vetor e .y o objeto para testar

has_element(dados$Petal.Width, 1.0)
## [1] TRUE
has_element(dados$Species, 1.0)
## [1] FALSE
